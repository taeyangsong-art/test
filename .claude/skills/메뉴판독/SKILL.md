# 메뉴판독 — 이미지 → 표준 메뉴 데이터 → POS 등록 산출물

원격팀의 메뉴등록 수작업(사진을 눈으로 읽어 POS에 타이핑)을 없애기 위한 스킬. 요청 이미지를 **판독**해 표준 JSON을 만들고, `scripts/menu-transcribe.js` 로 **① 토스플레이스 상품등록 양식 CSV** 와 **② 웹 입력 체크리스트** 를 생성한다. 담당자는 타이핑 대신 **검수 + 업로드/입력**만 한다. (집계는 [[메뉴등록집계]], AS 파이프라인은 [[원격집계]] 참고)

## 전체 흐름
```
슬랙 요청(이미지+텍스트) ──(Claude 비전 판독)──▶ 표준 JSON ──(menu-transcribe.js)──┬─▶ ① 토스플레이스 양식 CSV
                                                                                  └─▶ ② 웹 입력 체크리스트
                                                          menu-db.json(상품ID) ───┘  (변경·수정 완전자동화 전제)
```

## 실행 절차

1. **요청 수집** — 대상 메시지를 확보한다. 인자로 file_id/링크/매장명이 오면 그것을, 없으면 `slack_read_channel`(채널 `#oc팀_메뉴요청`, ID `C08740SFT1S`)에서 대상 스레드를 찾는다. 스레드의 첨부 이미지·요청문을 모두 모은다.

2. **이미지 판독** — 첨부 이미지는 `slack_read_file`(file_id)로 읽어 **비전으로 직접 판독**한다.
   - 메뉴 사진/손글씨/POS 관리자 화면 스크린샷 모두 대상. `카테고리 · 메뉴명 · 가격 · 옵션`을 정확히 읽는다.
   - 고객 소유 **Google Drive 링크 이미지**는 "generative AI ineligible"로 접근 불가할 수 있음 → 슬랙에 직접 첨부된 이미지를 우선 사용하고, Drive만 있으면 담당자에게 슬랙 첨부를 요청.
   - 자연어 지시("A→B로 변경", "옵션으로 …중 1개 선택", "바로 해주세요")를 구조화된 변경으로 해석한다.

3. **표준 JSON 작성** — 판독 결과를 아래 스키마로 임시 파일(scratchpad)에 저장한다.
   ```json
   {
     "store": "매장명", "biz": "사업자번호(하이픈 제거)", "pos": "토스플레이스",
     "request": "핵심 요청(감사·인사말 제외)",
     "changes": [
       { "op":"add",    "cat":"식사", "name":"비비막국수", "price":9000, "options":[] },
       { "op":"rename", "cat":"식사", "from":"막국수", "name":"물막국수", "price":9000 },
       { "op":"price",  "cat":"주류", "name":"생맥주", "price":5000 },
       { "op":"delete", "cat":"커피", "name":"배치브루 핫" }
     ]
   }
   ```
   - `op`: `add`(신규) · `rename`(이름변경) · `price`(가격수정) · `delete`(삭제). 가격은 숫자만(원).
   - **POS 불일치 주의**: 요청문의 포스명과 상호 뒤 VAN(KIS/KOCES 등)이 다르면 `pos`에 병기하고 보고 시 확인 요청.

   **3-B. 전체판독 모드 — "사진만 덜렁" 오는 경우** (예: "사진 넣어주세요~" + 메뉴판 사진만): `changes` 대신 `menu`(카테고리>항목) 로 **메뉴 전체를 통째로 추출**한다. 스크립트가 전 항목을 신규(add)로 펼친다.
   ```json
   {
     "store":"매장명", "biz":"사업자번호", "pos":"토스플레이스", "source":"image-only",
     "menu": [
       { "name":"식사", "items": [
         { "name":"눈꽃치즈닭갈비", "price":16000, "confidence":"high" },
         { "name":"얼큰막국수", "price":9000, "confidence":"low", "note":"가격 숫자 흐림" },
         { "name":"미나리닭갈비", "price":null, "confidence":"low", "note":"가격 잘림" }
       ]}
     ]
   }
   ```
   - **신뢰도(confidence) 필수 태깅**: 선명히 읽힌 항목 `high`, 흐림·애매·잘림은 `low` + `note`.
   - **추정 금지**: 가격이 안 보이면 지어내지 말고 `price:null`. 스크립트가 가격 미판독 항목을 **업로드 양식에서 자동 제외**하고 검수 목록으로 뺀다.
   - 여러 장이면 병합하고, 카테고리 구획(식사/사리/주류…)을 보존한다. 옵션(택1/추가금)은 `options` 에 담아 별도 표기.

4. **산출물 생성** — POS는 요청JSON의 `pos` 필드로 자동 판별(또는 `--pos`로 지정: `토스|퍼스트|오케이|스파로스`).
   ```
   node scripts/menu-transcribe.js "<요청JSON>" --db menu-db.json --out out
   ```
   - `out/<POS>_업로드_<매장>.csv` (①, add/rename/price 행) 와 `out/웹입력_체크리스트_<매장>.csv` (②, 전체) 생성.
   - **신규(add)** 는 상품ID 비워서 그대로 업로드 = **완전 자동**.
   - **변경·수정·삭제** 는 상품ID로 기존 상품을 매칭해야 안전(없이 이름만 바꾸면 중복 생성). `menu-db.json` 에 해당 매장 상품ID가 있으면 자동 채움, 없으면 `⚠️상품ID필요` 경고.

5. **보고 & 검수 요청** — 웹 체크리스트 표(작업/메뉴명/가격/**신뢰도/검수**/상품ID/비고)와 두 파일 경로를 한국어로 보고한다.
   - **검수 필요 목록**(`⚠️가격확인`=가격 미판독으로 양식 제외 · `⚠️판독확인`=저신뢰)을 별도로 강조해, 담당자가 **그 항목만 사진과 대조**하도록 한다. (high 항목은 그대로 통과 → 검수 리소스 최소화)
   - `⚠️상품ID필요` 가 있으면 "이 매장은 포스 상품목록 export 1회 필요"를 함께 안내한다.
   - 저화질/잘림으로 검수필요가 과도하면, 업로드 대신 **"메뉴판 전체가 선명한 사진 부탁드립니다"** 회신 초안을 제안한다.

## 메뉴 DB (상품ID) — 변경·수정 완전자동화의 전제
- `menu-db.json` 구조: `{ "<사업자번호>": { "<메뉴명>": "<상품ID>" } }`
- POS 상품목록을 1회 export(예: 토스플레이스 상품목록.xlsx 에 상품ID 실재)하면 그 매장의 변경·수정·삭제가 이후 완전 자동화된다.
- 신규 등록 시 발급된 상품ID를 DB에 축적하면 점진적으로 커버리지가 올라간다.

## 지원 POS 양식 (4종, `scripts/menu-transcribe.js` 의 `POS` 설정에 내장)
| POS | 컬럼 | 카테고리 | 가격 필드 | 매칭키 | 비고 |
|---|---|---|---|---|---|
| 토스플레이스 | 14 | 1단계(카테고리) | 기본가격 (원) | **상품 ID** ✅확정 | 상품ID 비우면 신규 |
| 퍼스트포스(KPN) | 28 | 1단계(대분류) | 판매가 | 상품코드(추정) | 기본값 낱개/재고Yes/KIOSK No |
| 오케이포스 | 22 | 1단계(대분류) | 판매단가 | **매핑상품코드** ✅확정 | 신규=매핑상품코드 비움·거래처 비움OK·원산지 필수(플레이스홀더 가능) |
| 스파로스포스 | 40 | **3단계(대>중>소)** | 판매단가 | ERP상품코드(추정) | ⚠️데이터 4행부터·영수증명·옵션내장 |

- 토스 외 3개는 **값 형식이 예시행 기반 추정**이므로, 산출물 헤더에 "(값형식 추정)"을 표시한다. 각 POS의 실제 채워진 export 샘플을 확보하면 `POS[<이름>].defaults`/`fields` 를 확정한다.
- 스파로스는 헤더를 생성하지 않고 **데이터 행만** 출력한다 → 공식 양식 4행부터 붙여넣기.
- 카테고리 계층: 스파로스는 `cat` 을 `"대>중>소"`(또는 `/`)로 주면 분할, 단일값이면 소분류로 들어간다.

## 주의
- 판독 정확도가 처리 품질을 좌우하므로, **애매한 항목(흐린 사진·손글씨·가격 누락)은 추정하지 말고 체크리스트에 `확인필요`로 표시**해 담당자 검수를 유도한다.
- 옵션(선택지/추가금)이 있는 메뉴는 `options` 에 담되, 토스플레이스 옵션은 별도 등록 절차라 체크리스트에 옵션 작업을 분리 표기한다.
- 이 스킬은 **양식 생성까지** 담당한다. 실제 업로드(Excel POS)·입력(웹 POS)·삭제는 담당자 또는 후속 RPA 단계에서 수행.
