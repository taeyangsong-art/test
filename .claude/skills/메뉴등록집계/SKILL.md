# 메뉴요청 채널 → 대시보드 '메뉴등록' 자동 집계

슬랙 `#oc팀_메뉴요청` 채널(ID: `C08740SFT1S`)의 **오늘** 메시지를 읽어, 리액션 이모지로 담당자별 메뉴등록 완료건수를 집계하고 `menu-data.js` 를 갱신한다. 대시보드(`index.html`)의 `메뉴등록`(key: `menu`) 카테고리 원본 데이터가 된다.

이 채널은 `#0_원격_as_요청`(원격집계 스킬)과 **동일한 이모지 규약**을 쓰지만, 채널 전체가 '메뉴등록' 한 카테고리라는 점이 다르다. (참고: [[원격집계]])

## 채널 특성 (요청 메시지 유형)
요청은 4가지 형태로 들어오며, 처리 규약은 모두 같다.
1. **CX매니저 정형 폼** — `상호 / 사업자번호 / 핸드폰 / 주소 / 인입유형 / 공급유형 / 내용` (가장 많음)
2. **CX매니저 간이 폼** — `상호 / 오프·온라인 / 사업자번호 / 대표자명 / 연락처 / 주소 / 내용`
3. **접수 봇 (`메뉴 등록 요청이 접수되었습니다.`)** — `:fork_and_knife: 새 메뉴 요청` / `- 사업자번호 / - 매장명 / - 휴대폰 / - 메뉴 수정 / - 포스 / - 이미지`
4. **메일·문자 사진 릴레이 봇** — `:mailbox_with_mail: 새 메일 도착` 또는 `~사진 보내드립니다`. **상호/사업자번호 라벨이 없는 보조자료** → 집계에서 자동 제외(같은 요청 중복 방지).

> 처리팀(원격): **배선유·김규빈** 중심(그 외 원격팀원 가능). 접수 시 `MENU` 봇이 담당자를 멘션한다.

## 이모지 규칙
- **완료 담당자**: `원격규빈`=김규빈, `원격선유`=배선유, `원격성현`=심성현, `원격동욱`=김동욱, `원격현기`=김현기, `원격태양`=송태양, `원격기범`=김기범, `원격상원`=서상원, `원격민석`=최민석
- **카테고리**: `원격메뉴등록`(이 채널은 전 건 메뉴등록이라 담당자 이모지만 있어도 완료로 인정)
- **집계 규칙**: 상호 또는 사업자번호가 있는 요청 메시지 = 1건. `원격OOO`(완료 담당자)가 찍히면 해당 직원 메뉴등록 **+1 완료**.
- **확인필요(pending)**: `OOO_확인`/`OOO확인` 류만 찍히고 `원격OOO` 는 없는 건(확인만 하고 미완료) + 아무 반응 없는 건(미착수).
- **중복요청**: `중복`으로 시작하는 이모지(`중복2` 등) → 집계 제외.
- **같은 요청 중복 제거**: 같은 날 같은 사업자번호(없으면 상호)로 완료가 여러 번 잡히면 1건으로 합산. 상호·사업자번호 라벨이 없는 사진/문자/메일 릴레이 메시지는 처음부터 제외.
- `x` 이모지(취소/보류로 추정)는 완료·확인 이모지가 따로 없으면 pending(미착수)으로 남긴다. 의미가 모호하면 처리팀에 확인 후 규칙 반영.

## 실행 절차

1. **오늘 날짜 확인** — 시스템 오늘 날짜를 `YYYY-MM-DD`로 확정한다.

2. **채널 메시지 읽기** — `slack_read_channel`(channel_id `C08740SFT1S`, `response_format: detailed`, `limit: 100`)로 최신 메시지를 읽는다.
   - 결과가 커서 파일로 저장되면 그 **파일 경로**를 확보한다.
   - 반환된 가장 오래된 메시지가 아직 **오늘**이면 `cursor`로 다음 페이지를 더 읽어 어제로 넘어갈 때까지 반복한다. (여러 파일이면 순서대로)

3. **집계 + 데이터 파일 갱신** — 저장된 파일 경로와 오늘 날짜로 실행:
   ```
   node scripts/tally-menu.js "<슬랙읽기결과파일경로>" <YYYY-MM-DD> --write menu-data.js
   ```
   - `menu-data.js` 의 `days["<오늘>"]` 항목을 추가/갱신하고 `version` 을 +1 한다. counts 는 `{ menu: { 담당자: 건수 } }` 구조(원격집계 slack-data.js 와 동일).
   - **여러 파일**(페이지네이션)이면 한 파일로 이어붙인 임시 파일에 대해 1회 실행한다. (완료는 사업자번호 기준 중복 제거, pending 은 상호+사업자 기준)

4. **검증** — 데이터 파일 문법 확인:
   ```
   node -e "global.window={};new Function('window',require('fs').readFileSync('menu-data.js','utf8'))(global.window);console.log('days:',Object.keys(global.window.MENU_DATA.days),'v',global.window.MENU_DATA.version)"
   ```

5. **요약 보고** — 완료 건수, 담당자별 완료 요약, 확인필요·미착수 목록(상호), 중복제외 건수, `menu-data.js` 누적 날짜 목록을 한국어 표로 정리해 보고한다.

## 주의
- 이모지 매핑에 없는 새 담당자 이모지가 보이면, 집계에서 누락됨을 보고하고 `scripts/tally-menu.js`의 `personMap` 에 추가를 제안한다.
- 데이터는 **`menu-data.js` 에만** 누적된다. 대시보드가 이 파일을 읽어 각 날짜의 `메뉴등록` 수치를 반영한다. (대시보드 연동은 별도 작업 — index.html 에서 `MENU_DATA.days[날짜].counts.menu` 를 `SLACK_DATA` 의 해당 날짜 counts.menu 로 병합)
- 같은 날 여러 번 실행하면 그 시점까지의 최신 실적으로 그 날짜 항목을 갱신한다.
- 원본 채널은 원격집계의 AS 채널과 **분리**되어 있으므로 두 스킬은 서로 다른 데이터 파일(`slack-data.js` vs `menu-data.js`)을 각각 소유한다.
